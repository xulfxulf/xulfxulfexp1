# HIRE-v2 版本一代码—文档一致性审计

## 审计结论

主结构与文档一致：

```text
CLIP ViT-B/16 全局观测；
RDE 风格图像与文本词元选择；
零初始化局部残差融合；
全局、细粒度、融合观测三套 SDM + ITC 监督；
融合观测用于训练期最佳检查点选择；
不使用支持包、概率身份后验或状态残差。
```

## 已修正问题

### 1. 文本无效词元污染 BatchNorm

原实现只在最大池化阶段屏蔽填充和特殊词元，但这些无效位置仍会经过两层感知机和 BatchNorm，可能改变运行均值与方差。修正后只有有效选中词元进入 MLP 与 BatchNorm，填充词元数值不会影响细粒度输出。

### 2. 文档文本词元数量定义不精确

文档现明确区分固定上下文候选预算与每条文本实际有效词元数量，并与修正后的实现一致。

### 3. LaTeX 公式控制字符损坏

两处 `\frac` 被错误写成控制字符。现已修复，并在静态审计中增加控制字符检查。

### 4. 梯度审计过弱

原审计只检查局部适配器梯度张量存在，未验证梯度非零。现改为检查梯度存在、有限且绝对值和大于零。

### 5. 损失聚合缺少单一真源

新增 `aggregate_anchor_objectives`，主模型和单元测试共同使用或验证该公式，避免代码与文档后续漂移。

### 6. 注意力适配器缺少等价性测试

新增合成（Transformer）前向等价测试，验证在仅把最后一层 `need_weights` 从关闭改为开启时，适配器输出与标准前向保持一致，同时成功返回最后一层注意力矩阵。

## 保留但明确说明的设计

总损失仍为：

```text
L = 0.5 × (L_global + L_local) + L_observation
```

这意味着全局、细粒度、融合观测的显式权重为 `0.5 / 0.5 / 1.0`。训练初始时融合观测等于全局观测，因此全局几何会收到两条梯度。这是有意的深监督锚定设计，并非重复求和错误，文档已明确说明。
