# HIRE-v2 版本二代码—文档一致性审计

## 审计结论

代码与设计文档的核心定义一致：

```text
版本一全局—局部完整观测路径保留；
版本二只增加身份随机效应；
支持集合只包含同身份、不同 image_id 的图像；
支持文本不进入数据流；
支持图不作为普通强正或弱正样本；
身份组至少需要两个有效支持图；
逐维不确定性与组内异质性共同构造可信交集；
严格对角身份组是唯一正目标；
相同 PID 非对角身份组被忽略；
身份辅助梯度与完整观测锚点隔离；
最终分数以有界身份残差修正版本一观测分数；
测试阶段不需要支持集合。
```

## 公式核对

工程损失为：

```text
sdm_loss
= 0.5 × (global_sdm + local_sdm) + final_sdm

itc_loss
= 0.5 × (global_itc + local_itc) + final_itc

identity_group_loss
= 0.1 × identity_group_nce
```

对应文档：

\[
L_{V2}=L_{anchor}+L_{final}+0.1L_{group}.
\]

身份最终分数训练形式：

\[
S_{final}=S_O+\alpha(S_{id}-\operatorname{sg}(S_O)).
\]

推理拼接表示的点积严格等于其数值形式：

\[
(1-\alpha)S_O+\alpha S_{id}.
\]

## 锚点保护核对

```text
主批次完整观测仍由版本一目标端到端更新；
身份均值读取 observation.detach()；
支持图 CLIP/TSE/fusion 路径在 no_grad 下执行；
支持图经过图像 TSE 时临时使用 eval 模式，避免 BatchNorm 运行统计被支持包改写；
身份组辅助损失只更新身份映射和不确定性头；
最终检索损失对观测主路径保留完整梯度。
```

## 留一关系核对

```text
支持图 pid 必须等于 anchor pid；
支持图 image_id 必须不同于 anchor image_id；
同一支持集合内 image_id 不得重复；
每个查询只把自己的对角身份组作为正目标；
同 PID 非对角身份组被掩码，不会因包含查询原图而产生隐式泄漏；
不同 PID 有效身份组作为负样本。
```

## 动态支持核对

支持选择由以下量决定：

```text
固定随机种子；
当前 epoch；
anchor image_id。
```

同一张图的两条文本共享同一个身份支持集合，不同训练轮次会轮换支持图。
